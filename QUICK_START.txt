┌─────────────────────────────────────────────────────────────┐
│            COMMON ERRORS & QUICK FIXES                      │
└─────────────────────────────────────────────────────────────┘

ERROR: "Port 5000 already in use"
FIX:   npx kill-port 5000
       npm start

ERROR: "Cannot connect to MongoDB"
FIX:   mongosh (should connect without error)
       Or: net start MongoDB (Windows)

ERROR: "Products not showing"
FIX:   npm run seed (run in backend folder)
       Then refresh browser

ERROR: "Order not saving"
FIX:   Check MongoDB is running (mongosh)
       Make sure all fields filled (name, phone, location)
       Check browser F12 console for errors

ERROR: "Cannot allow location"
FIX:   Click lock icon in address bar
       Set Location → Allow
       Try again

┌─────────────────────────────────────────────────────────────┐
│            LOCAL STORAGE (BROWSER)                          │
└─────────────────────────────────────────────────────────────┘

Cart is stored in browser's localStorage:
  Key: "cart"
  Format: Array of items with productId, name, price, quantity

To clear cart (in browser console):
  localStorage.removeItem('cart')
  location.reload()


┌─────────────────────────────────────────────────────────────┐
│            GEOLOCATION FLOW                                 │
└─────────────────────────────────────────────────────────────┘

User clicks "Get My Live Location":
  1. Browser requests permission
  2. User clicks "Allow"
  3. Gets latitude & longitude
  4. Shows Google Maps embed
  5. Coordinates stored when order submitted
  6. Admin can click "View on Google Maps"
  7. Opens Google Maps with location marker

NO API KEY NEEDED - Google Maps embed works without auth!

┌─────────────────────────────────────────────────────────────┐
│            TROUBLESHOOTING BY SYMPTOM                       │
└─────────────────────────────────────────────────────────────┘

SYMPTOM: Blank page when opening http://localhost:5000
CAUSE:   Backend not running
ACTION:  • Check terminal 1 is running npm start
         • Verify "Server running" message appears
         • Try http://localhost:5000 again

SYMPTOM: Products list empty in shop
CAUSE:   Database not seeded
ACTION:  • npm run seed (in backend folder)
         • Check for "products added" message
         • Refresh browser

SYMPTOM: Cart items not appearing on cart page
CAUSE:   localStorage not working
ACTION:  • Check browser console (F12)
         • Make sure cookies enabled
         • Try in different browser

SYMPTOM: Location button does nothing
CAUSE:   Permission denied or geolocation disabled
ACTION:  • Click lock icon in address bar
         • Set Location to "Allow"
         • Check site settings
         • Allow notifications permission

SYMPTOM: Cannot place order / error message
CAUSE:   Missing fields or server error
ACTION:  • Fill all form fields
         • Check backend terminal for errors
         • Check browser console (F12)
         • Restart backend if needed

SYMPTOM: Admin page shows no orders
CAUSE:   No orders placed or API error
ACTION:  • Place a test order first
         • Check backend is running
         • Check browser console (F12)
         • Refresh admin page
